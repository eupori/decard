## 1. Executive Summary

### 현재 제품의 핵심 문제

> **Decard는 "카드 생성기"이지, 아직 "학습 앱"이 아니다.**

PDF→근거 포함 카드 자동생성이라는 유일무이한 차별점이 있지만, **SRS(간격 반복)와 리텐션 루프가 없어서 "1회용 변환 도구"에 머물 위험**이 가장 크다. 4명의 사용자 패널 중 3명이 SRS 부재를 최대 단점으로 꼽았고, 전 C-Level이 이를 최우선 과제로 합의했다.

### 가장 긴급한 3가지

| 순위 | 과제 | 이유 |
|------|------|------|
| **1** | SRS (간격 반복) 구현 | 학습 앱의 본질. 없으면 리텐션 0, 유료화 근거 없음 |
| **2** | 무료 사용 제한 + 결제 인프라 | 수익화 퍼널의 전제조건. 현재 모든 기능이 100% 무료 |
| **3** | 데이터 수집 체계 (이벤트 로깅) | PMF 측정 불가. 데이터 없이는 의사결정 불가 |

### 2.2 기술 부채 (CTO + CIO)

| 영역 | 현재 | 한계 | 심각도 |
|------|------|------|--------|
| DB | SQLite (파일 기반) | 동시 쓰기 1개, 10명 동시 생성 시 DB lock | 🔴 높음 |
| 서버 | EC2 t3.small 단일 (2GB RAM) | 동시 3명 카드생성이 실질 한계 | 🔴 높음 |
| AI 호출 | Claude CLI subprocess | 메모리 비효율, API 직접 호출 대비 ~30% 낭비 | 🟡 중간 |
| 백그라운드 작업 | asyncio.create_task | 서버 재시작 시 태스크 소멸, 추적 불가 | 🟡 중간 |
| DB 백업 | **없음** | 서버 장애 시 전체 데이터 소실 | 🔴 **즉시 해결** |
| 테스트 코드 | 없음 | QA는 수동 시뮬레이션에 의존 | 🟡 중간 |
| 모니터링 | psutil 메모리만 | APM/에러 트래킹/로그 집중화 없음 | 🟡 중간 |
| 개인정보처리방침 | **없음** | Google Play 등록 불가, 법적 리스크 | 🔴 **즉시 해결** |

### 2.3 UX 이탈 포인트 (CXO + UI/UX)

```
[PDF 선택] → [업로드] → [대기] → [검수] → [학습] → [보관] → [재방문]
                         ★★★              ★★               ★★★
                       가장 큰           검수 의미         재방문 동기
                       이탈 포인트       불명확            부재
```

**핵심 문제:**
- **Time to Value = 3분** (PDF 선택 → 업로드 → 대기 → 카드 확인). 목표: 30초
- **대기 시간 불안감**: 1~5분 대기 중 "진짜 되고 있나?" → 중간 카드 미리보기 없음
- **검수 단계 혼란**: AI 자동채택 후 왜 또 확인해야 하는지 불명확
- **재방문 동기 부재**: 알림/SRS/스트릭/통계 전무 → 시험 끝나면 삭제

**즉시 개선 가능한 것들:**
| 개선 | 효과 | 난이도 |
|------|------|--------|
| 샘플 PDF 원클릭 체험 | Time to Value 3분→30초 | 낮음 |
| 학습 스트릭 (연속 학습일) | DAU 리텐션 핵심 | 낮음 |
| "알았다/몰랐다" 셀프 평가 | SRS 전 단계, 학습 효과 체감 | 낮음 |
| 목업 로그인 버튼 제거 | 신뢰도 향상 | 최저 |

### 2.4 사용자 패널 종합 피드백

| 페르소나 | 열광도 | 가장 원하는 것 | 결제 의향 |
|----------|--------|---------------|-----------|
| **박준영 (간호학과 대학생)** | ★★★★★ | SRS + 이미지 카드 | ₩14,900/월 (SRS 조건) |
| **이소현 (자격증 준비생)** | ★★★★ | SRS + 퀴즈렛 대체 | ₩6,900~14,900/월 |
| **김하늘 (고등학생)** | ★★★ | 사진→카드 + 테스트 모드 | ₩2,900~3,900/1주 |
| **정민수 (외국어 학습자)** | ★★ | TTS + 공유 덱 + 게임 | ❌ (현재 비타겟) |

**핵심 발견:**
- **가장 뜨거운 타겟 = PDF 강의자료가 있고 시험 대비가 급한 대학생** → 사업계획서 타겟 정확
- **외국어 학습자는 후순위** → 기존 세트 활용이 더 효율적, PDF→카드 니즈 약함
- **SRS 없이는 4명 중 3명이 "한 번 쓰고 끝"** → 유료 전환 불가능

## 3. 수익화 전략

### 3.1 가격 모델 (CEO + COO 합의)

**추천: "Freemium + 시즌 패스" 하이브리드**

| 티어 | 가격 | 내용 | 타겟 |
|------|------|------|------|
| **Free** | 0원 | 월 3~5회 생성, 기본 학습, SRS 기본 | 체험/유입 |
| **시험 패스 (14일)** | ₩4,900 | 무제한 생성, 전체 템플릿, 상세 통계 | 시험기간 단기 집중 |
| **학기 패스 (4개월)** | ₩14,900 | 무제한 + 고품질 AI(Sonnet) | 다과목 대학생 |
| **연간** | ₩39,900 | 전체 기능, 우선 지원 | 자격증 장기 준비생 |

**가격 근거:**
- ₩4,900: 검증된 앵커 ("4,900원이면 무조건 쓰죠"), 족보 가격대와 일치
- 14일: 시험기간 = 보통 2주, 사용자 행동 패턴에 정확히 맞는 단위
- 월 구독 지양: 대학생은 "매달 나가는 돈"에 극도로 민감. 기간제 패스가 심리적 저항 낮음

**Free vs Premium 경계:**

| 기능 | Free | Premium |
|------|------|---------|
| 카드 생성 | 월 3~5회 | 무제한 |
| 템플릿 | 1종 (정의형) | 3종 전체 |
| SRS 복습 | ✅ (핵심!) | ✅ |
| 학습 통계 | 기본 | 상세 (그래프, 기억률) |
| PDF 크기 | 5MB, 50페이지 | 10MB, 100페이지 |
| AI 모델 | Haiku (기본) | Sonnet (고품질) |
| CSV 내보내기 | ✅ | ✅ |

**핵심 원칙:**
- SRS와 기본 학습은 Free에 포함 (리텐션 = 전환의 전제조건)
- 생성 횟수 제한이 자연스러운 업셀 포인트
- CSV 내보내기는 Free 유지 (데이터 락인 = Quizlet의 실수 반복 안 함)

### 4.2 사용자 확보 로드맵

**100명 (콘시어지, ~1개월)**
- 지인 네트워크 15~20명
- 에브리타임 1:1 DM + 과목별 게시판 50~80명

**1,000명 (초기 성장, 1~4개월)**

- **인스타/틱톡**: "3시간 vs 5분" 변환 영상 → 200~300명
- **입소문**: 바이럴 계수 k=2~3 → 200~300명

**10,000명 (스케일업, 4~12개월)**
- Google Play ASO + 블로그 SEO → 3,000~5,000명
- 대학 제휴 + 유튜버 협찬 → 3,000~5,000명

## 5. 성장 전략 (CSO)

### 5.1 세그먼트 확장 경로

```
1단계: 수도권 대학교 문과대 (3~6개월)
  → 암기 비중 높은 과목 (교육학, 심리학, 간호학, 사회학)

2단계: 지방 거점 대학교 + 자격증 (6~12개월)
  → 에브리타임 학교별 게시판 확장
  → 한국사, 공인중개사, 간호사 필기시험

3단계: 고등학생 + B2B (12~24개월)
  → 사탐/과탐 내신 대비
  → 대학교/학원 제휴 (교수→학생 카드 배포)
```

### 5.2 방어 가능한 해자 (Moat)

| 해자 | 방어력 | 설명 |
|------|--------|------|
| 근거 포함 카드 | 중간 | 경쟁사 복제 가능하나, 프롬프트+파싱 노하우가 품질 차이 |
| 한국어 네이티브 최적화 | 높음 | 글로벌 기업이 한국 시험 문화 깊이 이해하기 어려움 |
| 학교별 누적 데이터 (장기) | 시간↑ | "○○대 교육학 최적 카드셋" → 네트워크 효과 |
