# 데카드 (Decard) — 사업 계획서

> PDF 올리면 시험 대비 끝. 근거 포함 암기카드 자동 생성기.

**작성일:** 2026-02-18
**버전:** v0.1 (MVP 기준)

---

## 1. 한 줄 요약

어떤 과목의 자료(PDF)를 올리든, 시험 대비에 바로 쓸 수 있는 암기카드 덱을 생성하고, 각 카드에 **근거(페이지/원문)**를 붙여 검수 비용을 최소화하는 학습 도구.

---

## 2. 문제 정의

### 학생들이 겪는 진짜 불편

1. **카드화 작업이 너무 귀찮다**
   - 시험 범위가 PDF/슬라이드로 주어지는데, 암기/복습 구조로 바꾸는 데 몇 시간씩 소모
   - 대부분 "귀찮아서 안 함" → 벼락치기로 직행

2. **GPT로 만들 수는 있지만 지속이 안 된다**
   - 복붙 → 정리 → 형식 맞추기 → 중복 제거 → 검수까지 결국 사람이 해야 함
   - "GPT는 카드 '생성'은 되는데, 근거/검수/덱 구조화는 매번 사람이 해야 해서 결국 귀찮아서 안 쓰게 된다"

3. **카드가 틀리면 신뢰가 무너진다**
   - AI가 만든 카드가 맞는지 확인하려면 다시 원문으로 돌아감
   - 검수 시간이 카드 만드는 시간보다 오래 걸리는 역설

### 우리가 해결하는 것

카드 '생성' 자체보다, **생성 이후의 검수 시간**을 줄여준다.
→ 각 카드에 근거(페이지+원문)를 붙여서 "이 카드가 맞나?"를 **5초 안에 판단** 가능하게.

## 3. 타겟

### 핵심 타겟

| 세그먼트 | 특성 | 사용 시점 |
|----------|------|-----------|
| **대학생** | 암기/정의/분류 비중 높은 과목 (인문, 사회, 교육, 간호 등) | 중간/기말 1-2주 전 |
| **고등학생** | 내신 시험, 수능 암기과목 (사회탐구, 과학탐구, 국어 문학) | 시험 전 + 수능 시즌 |
| **자격증 준비생** | 필기시험 (한국사, 공인중개사, 간호사, IT 자격증 등) | 상시 (시험 일정별) |

### 타겟 확장 로드맵

```
MVP: 대학생 (시험 대비)
 → 고등학생 (내신/수능)
 → 자격증 (필기시험)
 → 외국어 학습 (단어/문법 카드)
 → 직장인 (업무 지식 정리)
```

### 사용자 특성

- **벼락치기 패턴:** 시험 전주에 집중 사용 (계절성 높음)
- **가격 민감:** 5,000원 이하 = 충동 구매, 10,000원 = 고민, 20,000원 = 진지한 검토
- **GPT 기본 도구:** 무료 GPT를 이미 학습에 활용 → 차별점이 명확해야 함
- **족보 구매 문화:** 기출문제에 돈을 내는 행동이 이미 존재 → 결제 저항 낮음

## 4. 핵심 차별점 (3가지)

### 4.1 근거 포함 카드 (Verifiable Cards)

- 모든 카드에 **페이지 번호 + 원문 스니펫** 자동 첨부
- "이 카드가 맞나?" → 원문 기반으로 5초 안에 판단
- 불확실하면 카드를 안 만든다 (품질 우선)

### 4.2 시험형 템플릿 (Exam-native Templates)

| 템플릿 | 설명 | 예시 |
|--------|------|------|
| **정의형** | 'OO란?' | "피아제의 감각운동기란?" |
| **빈칸형 (Cloze)** | 핵심 문장 빈칸 | "감각운동기는 출생부터 약 ___세까지로..." |
| **비교형** | A vs B | "동화 vs 조절의 차이점은?" |

### 4.3 검수 워크플로 (Review-first)

- **채택 / 삭제 / 수정** 버튼 중심의 검수 UX
- 목표: '정확도 100%'가 아니라 **'검수 비용 최소화'**

## 5. 제품 컨셉

### 핵심 플로우

```
1. PDF 업로드 (강의자료/교재/필기)
2. 템플릿 선택 (정의/빈칸/비교)
3. AI가 근거 포함 카드 생성 (~15초)
4. 미리보기 5장 즉시 제공 (가치 체감 포인트)
5. 전체 카드 검수 화면 (채택/삭제/수정)
6. 다운로드 (CSV → Anki 임포트 가능)
```

### 카드 출력 포맷

```
[앞면] 피아제의 감각운동기란?
[뒷면] 출생~2세, 감각과 운동을 통해 세계를 탐색하는 시기
[근거] (p.45) "감각운동기는 출생부터 약 2세까지로, 영아는 감각과 운동을 통해..."
[태그] chapter:Piaget, type:정의, difficulty:easy
```

### 코스(과목) 단위 구조 (MVP 이후)

- 코스 = 과목별 공간 (자료, 덱, 태그, 이력을 묶음)
- MVP에서는 코스 개념 없이 단일 업로드 → 카드 생성으로 시작
- 검증 후 코스 시스템 도입

### 유료화 가설 (검증 후 적용)

| 플랜 | 가격 | 내용 | 타겟 |
|------|------|------|------|
| **Free** | 0원 | 월 3회 생성, 템플릿 1종 | 체험/유입 |

| **코스 패스 (30일)** | 6,900원 | 1과목 무제한, 템플릿 3종 | 자격증/장기 |

### 가격 결정 근거

- 대화 피드백: "4,900원이면 무조건 쓰죠"
- 족보 가격대: 5,000~10,000원 (기존 지불 행동과 일치)
- 코스 패스 → Unlimited 업셀 경로 설계

| 낙관적 | 구독 위주 14,900원 | ~34명 |

## 7. 경쟁 분석

| 대체재 | 강점 | 약점 | 데카드 차별점 |
|--------|------|------|---------------|
| **ChatGPT** | 무료/즉시 | 덱 구조화·근거·검수 없음 | 워크플로 자동화 + 근거 |
| **Anki 직접 제작** | 무료/강력 SRS | 만드는 시간 막대 | PDF→카드 자동화 |
| **퀴즈렛** | 쉬운 UI | 근거 없음, PDF 워크플로 없음 | 근거 기반 검수 |
| **노트가든 (종료)** | 비슷 컨셉 | 없어짐 | 시장 빈자리 차지 |
| **족보** | 실제 기출 | 과목별 비용, 불법 리스크 | 합법적 개인화 도구 |

### "GPT로도 되는데 왜 돈 내?" 에 대한 답

> GPT는 카드 '생성'은 되지만, 근거 확인/검수/덱 구조화는 매번 사람이 해야 해서 결국 귀찮아서 안 쓰게 된다. 데카드는 이 전체 과정을 1분 안에 끝낸다.

## 8. 리스크 & 대응

### 기술 리스크

| 리스크 | 심각도 | 대응 |
|--------|--------|------|
| PDF 파싱 품질 (표/수식/스캔) | 높음 | 텍스트 PDF만 우선, OCR은 후순위 |
| AI 비용 통제 | 높음 | 모델 라우팅 + 월 크레딧 + 캐싱 |
| 카드 정확도 (환각) | 높음 | 근거 없으면 미생성 + 검수 UX |

### 사업 리스크

| 리스크 | 심각도 | 대응 |
|--------|--------|------|
| GPT 대체 인식 | 최고 | 검수 워크플로 차별 강조 |
| 시즌성 매출 | 높음 | 자격증/외국어로 비시험기간 커버 |
| 1인 운영 한계 | 중간 | 자동화 최대화 |
| 저작권 이슈 | 중간 | 공유 기능 금지, 파일 자동 삭제 |

## 9. 90일 로드맵

| 주차 | 마일스톤 | 핵심 활동 |
|------|----------|-----------|
| **1주 (Day 1-3)** | MVP 출시 | PDF→카드→검수→다운로드 핵심 플로우 |
| **2주** | 콘시어지 테스트 | 10-15명에게 배포, 인터뷰, 데이터 수집 |
| **3-4주** | MVP 보강 | 피드백 반영, 빈칸형 추가, UX 개선 |
| **5-6주** | 유료화 테스트 | Free + 코스 패스 도입, 결제 의향 확인 |
| **7-9주** | 앱 개발 | React Native 또는 Flutter, 학습 모드 특화 |
| **10-13주** | 성장 | 마케팅, 템플릿 고도화, 포맷 확장 |

### Go/No-Go 기준

| 판정 | 조건 |
|------|------|
| **Go** | 10명 중 3명 이상 결제 의향 + 2명 이상 반복 사용 |
| **Pivot** | 근거/검수가 차별로 인식되지 않지만 카드 자체 니즈는 있음 |
| **No-Go** | "GPT로 할게"가 과반 + 반복 사용 없음 |
