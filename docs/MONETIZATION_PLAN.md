## Executive Summary

> **"쓴 만큼만 낸다. 구독은 없다."**

Decard의 수익화는 **생성권 묶음(소모품형)** + **기능 해금 하이브리드**로 확정한다.
7개 페르소나(대학생/고등학생/자격증/CFO/수익화전문가/심리학자/GPT파워유저) **만장일치**로 기간제 구독 대신 1회성 생성권을 선택했다.

### 확정 모델: 생성권 + 시험기간 패스

```
무료          월 3회 생성 (매월 리셋)
──────────────────────────────────────
스타터        5회    ₩1,100   (유효기간 없음)
스탠다드     15회    ₩4,900   (유효기간 없음)
──────────────────────────────────────
시험기간패스  무제한  ₩9,900/30일 + 프리미엄 기능 해금
```

### 이 모델을 선택한 3가지 이유

| # | 이유 | 근거 |
|---|------|------|
| 1 | **₩1,100 첫 결제 허들 파괴** | 구독 ₩4,900은 "고민", ₩1,100은 "충동". 전환율 2배 이상 차이 (5~8% vs 12~20%) |
| 2 | **AI 비용 통제 = 생존** | 기간제 무제한은 헤비유저 1명이 ₩7,000 적자. 생성권은 판매 시점에 마진 확정 |
| 3 | **학생 소비 패턴과 일치** | 시험기간 폭발 + 방학 휴면. 구독은 이 패턴과 충돌. "쓸 때만 산다"가 자연스러움 |

### 유료화 전제조건: 품질 개선 필수

> **"횟수제로 가는 이상 한 번의 퀄리티가 굉장히 좋아야 한다. 현재 퀄리티로는 돈 내기 아까워."**

유료화 이전에 반드시 해결할 것:
1. 카드 양 증가 (30p PDF → 현재 20~30장 → 목표 50~80장)
2. OX형 템플릿 추가 (프롬프트만 수정)
3. SRS 구현 (리텐션 = 유료 전환의 전제)

## 1. 한국 학생 지출 패턴 분석

### 1.1 교육 앱 지출 현황

| 항목 | 가격대 | 학생 반응 |
|------|--------|-----------|
| 족보 (대학별) | ₩3,000~5,000/과목 | "시험 전엔 무조건 삼" |
| 인강 (메가스터디 등) | ₩10만~30만/과목 | "부모님이 결제" |
| 앱 구독 (노션, 굿노트) | ₩5,000~8,000/월 | "필수 도구만 결제" |
| Quizlet Plus | ₩7,900/월 (~₩48,000/년) | "비싸서 무료로 버팀" |
| ChatGPT Plus | ₩28,000/월 | "학생에겐 부담, 무료 충분" |

### 1.2 결제 심리학

- **₩1,100 = 편의점 음료**: 결제 행위 자체를 "고민"이 아닌 "충동"으로 만듦
- **구독 피로 포화**: 넷플릭스, 유튜브, 멜론... "월 고정 지출" 심리 계좌는 이미 꽉 참 (Thaler의 Mental Accounting)
- **1회성 = "변동 지출" 계좌**: 에너지 드링크, 족보처럼 "필요할 때 사는 것"으로 분류 → 저항 낮음
- **유효기간 없음**: 구매 리스크 제로 → "일단 사놓자" 심리 (Shu & Gneezy, 2010)

### 1.3 결제 수단

| 순위 | 수단 | 비율 | 비고 |
|------|------|------|------|
| 1 | Google Play 인앱결제 | 1순위 | 앱스토어 표준 |
| 2 | 카카오페이 | ~45% | 웹 결제 시 |
| 3 | 토스페이 | ~25% | 20대 급성장 |

## 2. GPT 위협 분석 — "직접 하면 되잖아?"

### 2.1 GPT 파워유저의 실제 워크플로우 (Devil's Advocate 검증)

```
GPT Plus 구독자가 PDF→카드를 직접 만드는 과정:
1. PDF를 GPT-4o에 업로드 — 1분
2. 프롬프트 작성 ("시험 카드 만들어줘") — 1분
3. 결과 대기 — 1~2분
4. TSV 복사 → Anki 임포트 — 5~10분
5. 포맷 깨짐 수정, 태그 정리 — 5~10분
6. 긴 PDF는 청크 나눠서 반복 — ×2~3회
──────────────────────────────
총 소요: 15~30분 (매번)

문제점:
- 50페이지 넘으면 뒷부분 짤리거나 요약됨
- 페이지 번호 부정확 (환각)
- 품질 편차 큼 (같은 프롬프트인데 어떤 날은 20장, 어떤 날은 8장)
- Anki까지 가야 SRS 학습 가능 (그 세팅 자체가 진입장벽)
```

### 2.2 Decard vs GPT

| 차원 | GPT 직접 | Decard |
|------|---------|--------|
| 시간 | 15~30분/PDF | **2~5분/PDF** |
| 근거/출처 | 부정확 (환각) | **pdfplumber로 정확한 페이지** |
| 출력→학습 | 별도 Anki 임포트 | **원스톱 (생성→검수→SRS)** |
| 비용 | GPT Plus ₩28,000/월 | **₩1,100/5회** |
| 일관성 | 프롬프트마다 다름 | **표준화된 템플릿** |

### 2.3 GPT 유저 세그먼트별 전략

| 유저 타입 | 비율 | 전략 |
|-----------|------|------|
| GPT 초보 (PDF→카드 프롬프트 자체를 모름) | 70% | "원클릭" 편의성으로 흡수 |
| GPT 중급 (카드는 만들지만 학습 도구 없음) | 20% | SRS + 보관함 + 근거가 차별점 |
| GPT 고급 (자체 파이프라인 구축) | 10% | 타겟 아님. 포기 |

### 2.4 방어 가능한 해자 vs 방어 불가능한 영역

**방어 가능 (진짜 해자)**:
- 생성→검수→SRS→시험모드 원스톱 루프
- Anki 모르는 비개발자 대학생 90%가 진짜 시장
- 한국어 시험 문화 특화

**방어 불가능 (솔직한 인정)**:
- GPT-5가 "PDF→Anki 원클릭 내보내기" 기능 추가 시 생성 가치 감소
- 대응: 카드 "생성"은 입구일 뿐. **학습 생태계가 진짜 제품**

## 5. 품질 요구사항 — 유료화 전 필수

### 5.1 현재 문제와 목표

> **"횟수제로 가는 이상, 1회 ₩220~₩327을 낼 가치가 있어야 한다."**

| 항목 | 현재 | 목표 | 해결 방법 |
|------|------|------|-----------|
| 카드 수 (30p 기준) | 20~30장 | **50~80장** | MAX_CARDS 50→120, 페이지당 4~7장 |
| 템플릿 | 정의/빈칸/비교 3종 | **+OX형, 사례적용형** | 프롬프트 추가 |
| 체감 가치 | "이게 다야?" | **"족보 안 사도 되겠다"** | 양 확보 + 시험 모드 |

### 5.2 "돈 낼 만한" 최소 산출물 기준

**1회 생성 (₩220~327)에서 유저가 기대하는 것:**

| 기준 | 최소 수준 |
|------|-----------|
| 카드 수 | 30p PDF → 최소 40장 이상 |
| 시험 적중률 | 카드 중 30% 이상이 "시험에 나올 수 있겠다" 수준 |
| 즉시 학습 가능 | 검수 없이 바로 학습 시작 가능한 완성도 |
| 근거 정확도 | 페이지 번호 정확 (GPT 대비 핵심 차별점) |

**"족보 대체" 수준:**
- 족보 = 기출문제 20~30문제 + 답
- Decard 목표 = 40장 카드 + 시험 모드(모의고사) + AI 채점

### 5.3 즉시 실행 가능한 품질 개선

| 순서 | 작업 | 효과 | 난이도 |
|------|------|------|--------|
| 1 | MAX_CARDS 50→120, 페이지당 4~7장 | 카드 양 2~3배 증가 | **프롬프트 수정만** |
| 2 | OX형 템플릿 추가 | 카드 다양성 + 양 증가 (1개념→OX 2~3문제) | **프롬프트 추가만** |
| 3 | 난이도 태그 활용 (이미 존재) | "시험 전날용/심화학습" 필터 | 프론트 UI만 |

## 7. 페이월 UX 설계

### 7.1 Aha Moment

> **"내 PDF에서 카드가 자동으로 나온 순간"** = 반드시 무료로 경험시켜야 함

- 첫 생성은 절대 막지 않는다
- 3회면 "이거 되네" 판단에 충분 (1회 테스트 + 2회 실사용)
- Aha Moment 전에 결제 요구 = 사망

### 7.2 Soft Paywall 원칙

| ❌ 하지 말 것 | ✅ 해야 할 것 |
|--------------|--------------|
| 전면 팝업으로 가로막기 | 인라인 배너로 자연스럽게 안내 |
| "결제 안 하면 못 씀" 강압 | "무료 생성 N회 남았어요" 카운트다운 |
| 학습 기능 잠금 | 학습/SRS는 항상 무료 |
| 데이터 내보내기 차단 | CSV 내보내기 항상 가능 |
| 템플릿 잠금 | 3종 전부 무료로 |

### 7.3 페이월 메시지

| 시점 | 메시지 | 톤 |
|------|--------|-----|
| 1회 소진 | "이번 달 무료 생성 2회 남았어요" | 정보 전달 |
| 2회 소진 | "마지막 1회! 스타터 5회 ₩1,100으로 더 만들어보세요" | 부드러운 권유 |

| 콘텐츠 | 채널 | 효과 |

| "AI 카드가 진짜 시험에 나왔다" 후기 | 에브리타임 | 최고 |
| "족보 대신 이거 쓰세요" | 에브리타임 | 높음 |
| "GPT 30분 vs Decard 5분" | 유튜브 쇼츠 | 높음 |

### 8.4 입소문 구조

> "이거 무료로 3번 써보고, 괜찮으면 1,100원에 5번 더 써봐"

- 이 한 마디가 가능한 것이 생성권 모델의 최대 강점
- "무료 5번 쓰고 구독 끊어봐"는 대학생에게 부담스러움

### 8.5 리퍼럴 프로그램

```
추천한 사람: 무료 생성 +2회 (최대 월 9회)
추천받은 사람: 첫 달 무료 생성 +1회 (총 4회)

→ 바이럴 계수 k=1.5~2.0 목표
→ 별도 결제 인센티브 없음 (무료 횟수 확장만, 비용 통제)
```

## 부록: 토론 근거 — 기간제 구독을 폐기한 이유

7명 만장일치로 생성권을 선택한 핵심 근거:

| 관점 | 기간제 구독의 문제 | 생성권의 강점 |
|------|-------------------|--------------|
| 대학생 | "또 구독이야?" 거부감 | "₩1,100이면 고민 안 함" |
| 고등학생 | 부모 설득 필요 (₩4,900 구독) | 용돈으로 해결 (₩1,100) |
| 자격증 준비생 | 6개월 총비용 ₩34,500 | 총비용 ₩9,900 |
| CFO | 무제한 = 헤비유저 적자 | 판매=비용 확정 |
| 수익화 전문가 | 전환율 5~8% | 전환율 12~20% |
| 소비자 심리학자 | "월 고정 지출" 계좌 포화 | "변동 지출" 계좌 = 저항 없음 |
| GPT 파워유저 | "해지 안 하면 손해" 계산 시작 | "1회 ₩220 vs GPT 30분" = 납득 |
